KEY FEATURES
============

1. FILE UPLOAD
   - Drag-and-drop interface for S1 and S2 files
   - Support for .tif, .jp2, .nc formats
   - Batch processing capability
   - File validation

2. FLOOD DETECTION
   - Real-time model inference on satellite data
   - Pixel-level probability mapping
   - Adaptive confidence thresholds (0.3-1.0)
   - Green → Yellow → Orange → Pink color scale

3. RISK ASSESSMENT
   Combines two factors:
   - Flood Probability (70% weight)
   - Population Proximity (30% weight)
   
   Results in 4 risk levels:
   🟢 Low (< 25%)
   🟡 Medium (25-50%)
   🟠 High (50-75%)
   🔴 Immediate Action (> 75%)

4. INTERACTIVE MAPS
   - Color-coded heatmaps
   - Hover for exact probabilities
   - Blue contours for detected floods
   - Full zoom/pan controls

5. DETAILED STATISTICS
   - Flood coverage percentage
   - Average flood probability
   - Affected pixel count
   - Detection confidence metrics

6. AUTOMATED ALERTS
   - Critical alerts for high probability
   - Population proximity warnings
   - Contextual recommendations
   - Action guidance

7. HISTORY TRACKING
   - Upload timestamp log
   - Satellite source tracking
   - Flood probability trends
   - Risk level history


QUICK START
===========

1. Open FloodML Dashboard
2. Click "📡 Data Upload" in sidebar
3. Upload satellite file (.tif, .jp2, or .nc)
4. Set confidence threshold (default: 0.5)
5. Click "🔍 Process & Analyze"
6. Review interactive flood map
7. Check detailed statistics in expandable sections


RISK LEVELS EXPLAINED
=====================

🟢 LOW RISK
  - Flood probability < 25%
  - Remote location
  - Action: Continue monitoring

🟡 MEDIUM RISK
  - Flood probability 25-50%
  - Mixed/rural areas
  - Action: Increase monitoring

🟠 HIGH RISK
  - Flood probability 50-75%
  - Populated/coastal areas
  - Action: Prepare contingencies

🔴 IMMEDIATE ACTION
  - Flood probability > 75%
  - Urban/critical areas
  - Action: Emergency protocols


TECHNICAL SPECS
===============

Input Data:
  - Sentinel-1 SAR: VV/VH polarization bands
  - Sentinel-2 Optical: Multispectral imagery
  - Landsat 8/9: OLI/TIRS data
  - Format: .tif, .jp2, .nc

Processing:
  - Random Forest classifier (100 trees)
  - Model accuracy: ~94.7%
  - Feature extraction: Index computation + DEM
  - Post-processing: Morphological filtering

Performance:
  - Small file (< 50MB): 5-15 seconds
  - Medium file (50-200MB): 30-60 seconds
  - Large file (> 200MB): 60-120 seconds
  - Batch (5 files): 2-5 minutes
  - GPU acceleration: 50-70% faster (if available)


COLOR SCALE REFERENCE
=====================

Map Colors:
  Dark Green (#003a00):  Safe - No flood risk
  Neon Green (#00ff88):  Low - < 30% probability
  Yellow (#ffff00):      Medium - 30-50%
  Orange (#ff6600):      High - 50-70%
  Hot Pink (#ff0080):    Critical - > 70%

Blue Contours:
  Areas with flood probability > 50%


CONFIDENCE THRESHOLDS
====================

0.3 - 0.4:  Maximum sensitivity (emergency response)
            ⚠️ May include false positives
            ✓ Good for: Quick threat assessment

0.5:        Balanced (default recommended)
            ✓ Best for: Routine monitoring
            ✓ Good balance of sensitivity/specificity

0.7 - 0.8:  High specificity (verification)
            ⚠️ May miss weak signals
            ✓ Good for: Confirmation analysis


POPULATION DATA SOURCES
=======================

Worldpop:    Global population density (default)
             - Resolution: 100m
             - Coverage: Complete global

GHSL:        Global Human Settlement Layer
             - Resolution: 38m
             - Focus: Urban/settlement areas

None:        Auto-estimate from scene
             - Uses scene characteristics
             - No external data required


SATELLITE SOURCE COMPARISON
============================

Sentinel-1 (SAR):
  - Works in clouds (big advantage)
  - VV/VH bands for water detection
  - Resolution: 10m

Sentinel-2 (Optical):
  - Multi-spectral indices (NDVI, MNDWI)
  - Better vegetation mapping
  - Resolution: 10-60m (depends on band)

Landsat 8/9:
  - Free archive data
  - 30m resolution
  - Thermal bands available


TROUBLESHOOTING
===============

Problem: File won't upload
→ Check format (.tif, .jp2, .nc)
→ Verify file size < 2GB
→ Ensure file is not corrupted

Problem: Analysis taking too long
→ Try smaller files
→ Reduce scene resolution
→ Check system resources

Problem: No map displayed
→ Verify geospatial metadata in file
→ Check console for error messages
→ Try refreshing the page

Problem: Risk seems incorrect
→ Check confidence threshold setting
→ Verify population data source
→ Review scene for cloud cover

SUPPORTED FORMATS
=================

GeoTIFF (.tif, .tiff):
  - Most common format
  - Full metadata support
  - Recommended for Sentinel-1

JPEG 2000 (.jp2):
  - Compressed format
  - ESA standard
  - Recommended for Sentinel-2

NetCDF (.nc):
  - Scientific data format
  - Good for model outputs
